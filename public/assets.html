Comment les fichiers s'articulent

Une carte légère du dépôt, montrant comment les dossiers clés, les configurations et les ressources s'alimentent mutuellement. Suivez le flux depuis les points d'entrée jusqu'aux styles, composants UI et assets publics.

Entrée

Flux des requêtes

pages/ Routes, pages rendues côté serveur et endpoints API.
components/ Blocs UI réutilisables référencés par les pages.
styles/ Styles globaux et définitions de thème.
Ressources

Fichiers statiques

public/ Images, icônes et fichiers téléchargeables servis tels quels.
assets.html Cette page explicative, stockée avec le code.
Config

Règles du projet

next.config.ts Configuration de build et d'exécution pour Next.js.
tsconfig.json Paramètres TypeScript partagés dans l'app.
package.json Scripts et dépendances qui alimentent le projet.

Relations en un coup d'œil

Les pages importent des composants, les composants consomment les styles, et les assets vivent dans public pour un accès direct. Les fichiers de configuration pilotent la compilation et le service.

Pourquoi c'est important

Quand vous ajoutez une nouvelle page, vous créez ou réutilisez des composants, branchez les styles partagés et référencez des assets dans public. Un flux clair garde la HMR rapide et prévisible.
